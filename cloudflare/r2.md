# Cloudflare R2

R2 is Cloudflare's S3-compatible object storage with zero egress fees. Store files, images, backups, or any blob data and access it from Workers, the S3 API, or public URLs -- without paying to read your own data.

## Key Features

- **S3-compatible API** - Drop-in replacement for most S3 workflows
- **Zero egress fees** - No charge for data read out of R2
- **Workers binding** - Direct in-process access from Workers (no network hop)
- **Public buckets** - Serve files via custom domains or `r2.dev` URLs

## Local Uploads (Beta, Feb 2026)

Improves global upload performance by allowing uploads to the nearest Cloudflare data center rather than routing to a single region. Reduces latency for users uploading from far away.

## R2 SQL

Query structured data stored in R2 using SQL syntax.

**Approximate aggregation functions** (Feb 2026) - New functions for fast approximate counts and statistics on large datasets without scanning every row.

## Pricing

| | Free | Paid |
|---|---|---|
| Storage | 10 GB/month | $0.015/GB-month |
| Class A ops (writes) | 1M/month | $4.50/million |
| Class B ops (reads) | 10M/month | $0.36/million |
| Egress | Free | Free |

## Basic Usage

```toml
# wrangler.toml
[[r2_buckets]]
binding = "MY_BUCKET"
bucket_name = "my-bucket"
```

```typescript
// Write
await env.MY_BUCKET.put("key", "value");

// Read
const obj = await env.MY_BUCKET.get("key");
const text = await obj.text();

// List
const list = await env.MY_BUCKET.list({ prefix: "uploads/" });
```
